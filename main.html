Úvod do KBB & Malware

Typy hackerů

⬜ White Hat

Etičtí hackeři. Identifikují a opravují zranitelnosti se svolením organizace.

⬜ Gray Hat

Bez kriminálních úmyslů, ale nabourávají se bez souhlasu — obvykle aby upozornili na chybu.

⬛ Black Hat

Kyberzločinec. Jedná nelegálně pro vlastní prospěch.

Týmy etických hackerů

Red Team

Ofenzivní tým. Hledá zranitelnosti, provádí penetrační testování.

Blue Team

Defenzivní tým. Ochraňuje sítě, kontroluje logy, funguje jako tech support.

Purple Team

Zajišťuje komunikaci a spolupráci mezi Red a Blue týmem.

Druhy malwaru

Virus

Šíří se vkládáním do jiných spustitelných souborů či dokumentů. Bez vědomí uživatele.

Červ (Worm)

Šíří se přes sítě využitím zranitelností OS. Nepotřebuje hostitele.

Trojský kůň

Tváří se jako užitečný software (hra, nástroj), ale skrytě škodí.

Adware / Spyware

Zahlcuje reklamou / špehovává. Spyware: Keylogger (klávesy), Screenlogger (obrazovka).

Ransomware

Zašifruje disk a požaduje výkupné. Varianty: šifrovací (přístup zachován) a zamykací (blokuje vše).

Speciální typy malwaru

Polymorfický

Mění vlastní kód při každém běhu — obtížná detekce antivirem.

File-less

Existuje pouze v RAM. Nepíše soubory na disk — neviditelný pro klasické AV nástroje.

Analýza malwaru

Statická

Bez spuštění kódu. Nástroj: virustotal.com

Dynamická

Analýza chování při spuštění v izolaci. Nástroj: any.run

Ransomware skupiny

REvil

Zaniklá skupina — poskytovala RaaS (Ransomware as a Service).

Lazarus

Severokorejská státem sponzorovaná skupina.

Cyber Partisans

Běloruská hacktivistická skupina. Požadovala místo peněz politické cíle — propuštění 50 vězňů.

Soubory v Linuxu

Magic Numbers

Unix neurčuje typ souboru podle přípony, ale podle prvních bajtů (header) — tzv. magic numbers. Příkaz file.

Relativní vs absolutní cesta

Absolutní = celá cesta z /root. Relativní = cesta z aktuální složky. ~ = /home, - = předchozí složka.

Shell redirection

| piping, > přepis souboru, >> přidání na konec.

Oprávnění souborů

Struktura

Oprávnění pro: Vlastník · Skupina · Ostatní

r · w · x

read = 4, write = 2, execute = 1. Sčítají se → 7 = rwx, 6 = rw-, 4 = r--

Příklad
chmod 764 soubor
# 7 = rwx (vlastník)
# 6 = rw- (skupina)
# 4 = r-- (ostatní)

chmod u+rwx,g+rw,o+r soubor
# totéž, jinak zapsáno
"r,w,x / 4,2,1" — Nathan

Sítě & Síťová bezpečnost

Základní pojmy

LAN / WAN / Internet

LAN = lokální síť. WAN = propojení více sítí. Internet = všechny sítě světa.

Switch vs Router

Switch propojuje zařízení v LAN. Router směruje provoz mimo LAN (do WAN).

Server – Client – P2P

Server poskytuje služby klientům. P2P = bez centrálního serveru, zařízení sdílí navzájem.

Fyzická média

Měděný kabel (UTP/STP)

Přenáší elektrické signály. Koncovka RJ-45. UTP náchylnější na rušení, STP má stínění.

Optický kabel

Přenos světla — tenká křehká vlákna. Mnohem vyšší rychlosti a delší vzdálenosti.

Koaxiální kabel

Dnes již zastaralý.

Adresace

IPv4 (32-bit)

4 oktety oddělené tečkami. ~4.3 miliard adres → dochází. Privátní rozsahy: 10.x.x.x, 172.16–31.x.x, 192.168.x.x

IPv6 (128-bit)

Hexadecimální. 340 undecillionů adres. Postupná náhrada IPv4.

Loopback

127.0.0.1 (localhost) — zařízení posílá data samo sobě.

MAC adresa

Fyzická adresa síťové karty. 48-bit, hex formát. 1. část = výrobce, 2. část = karta. Příklad: 00:0a:95:9d:67:16

Maska sítě & Gateway

Maska určuje počet IP adres v síti. Gateway = adresa routeru pro provoz mimo LAN.

Protokoly

DNS

Překlad doménových jmen na IP. ssps.cz → 1.2.3.4. Hierarchie: TLD → doména → subdoména.

DHCP

Dynamické přidělování IP adres. Běží obvykle na routeru.

TCP vs UDP

TCP = spolehlivý (kontrola ztrát, pomalejší). UDP = rychlý, bez kontroly — streaming, DNS.

HTTP(S) / SSH / Telnet

HTTP port 80, HTTPS port 443 (šifrovaný). SSH port 22 (šifrované CLI). Telnet port 23 — zastaralý, nebezpečný.

ARP

Mapuje IP adresu na MAC adresu v rámci LAN.

Kryptografie

Šifrování

Symetrické

1 klíč pro šifrování i dešifrování. Rychlejší. Příklady: AES, Serpent, Twofish.

Asymetrické

2 klíče: veřejný (šifruje) + soukromý (dešifruje). Příklady: RSA, CRYSTALS-Kyber (post-kvantové).

Šifrování disku

LUKS (Linux standard, open-source). VeraCrypt (univerzální, all-OS, open-source).

Hash

Co je hash?

"Otisk prstu" dat. Jednosměrná funkce — nejde dešifrovat zpět. Neměnný a unikátní.

Příklady

MD5 (zastaralý), SHA-512, Blake2

Využití

Ukládání hesel, digitální podpisy, ověření integrity dat.

Ukázka MD5
ahoj → 79c2b46ce2594ecbcb5b73e928345492

Crackování hashů

Wordlisty, Rainbow Tables. Nástroje: Hashcat, John the Ripper.

Kódování & Obfuskace

Kódování (Encoding)

Přepis do jiné formy dle algoritmu. NENÍ šifrování! Příklady: Base64, Morseovka, Braillovo písmo.

Obfuskace

Znesnadnění čitelnosti kódu pro člověka. Využívá kódování, ale nemá vliv na funkci kódu.

Digitální podpis

Odesílatel podepisuje hash zprávy svým soukromým klíčem. Příjemce ověří veřejným klíčem.

Signal & Komunikace

Double Ratchet algoritmus

Rotace klíčů — každá zpráva má jiný klíč. Zaručuje forward secrecy (staré zprávy zůstanou tajné) a post-compromise security (bezpečnost se sama obnoví).

Diffie-Hellman

Algoritmus pro bezpečnou výměnu symetrických klíčů přes nezabezpečený kanál.

Post-kvantová kryptografie

Ochrana před kvantovými počítači. Ohrožena je hlavně asymetrická kryptografie (RSA).

Kryptoměny & Blockchain

Blockchain

Decentralizovaná databáze transakcí. Transakce jsou veřejně dohledatelné (Bitcoin, Ethereum).

Monero

Jediná známá kryptoměna s plnou anonymitou transakcí.

Využití v kyberkriminalitě

Ransomware skupiny požadují platby v kryptoměnách kvůli anonymitě.

Kybernetické útoky & OSINT

⚠️  Tento obsah slouží pouze pro vzdělávací účely. Útoky na cizí systémy bez povolení jsou nelegální. Dodržujte zákony své země.
Zranitelnosti

Exploit

Program nebo skript využívající programátorskou chybu v kódu.

CVE

Common Vulnerabilities and Exposures — identifikátor známé zranitelnosti. Př: log4shell = CVE-2021-44228

CWE

Kategorizace typů slabostí. Př: CWE-79 = XSS.

CVSS

Skóre závažnosti zranitelnosti (verze 4.0). Zahrnuje vektor útoku, komplexitu, dopad…

Zero-day

Exploit pro chybu, která není veřejně známá. Neexistuje patch.

MITRE ATT&CK

Framework pro analýzu útočných technik a APT skupin. → attack.mitre.org

Typy útoků

Phishing

Podvodné emaily imitující banku / službu. Cíl: hesla, kreditní karty. Poznáme podle podezřelé domény (např. @cs0b.cz místo @csob.cz). Nejčastější Initial Access (MITRE).

DoS / DDoS

Zahlcení serveru požadavky. DDoS využívá botnet (síť ovládaných PC s různými IP) — těžko blokovat.

XSS

Cross-Site Scripting — vložení JS kódu do webové stránky. Cíl: cookies, localStorage, přesměrování.

SQLi

SQL Injection — vložení kódu do SQL dotazu. Cíl: hesla a citlivá data z DB.

Sociální inženýrství

Manipulace lidí pro získání informací. Phishing je příkladem.
